Uma visão geral do JSP

Julho/2002   





JavaServer Pages (JSP) é uma especificação da Sun Microsystems que combina Java e HTML para fornecer conteúdo dinâmico para páginas Web. Quando a intenção é criar conteúdo dinâmico, JSPs são mais convenientes de se escrever do que servlets HTTP porque elas permitem que você encaixe código Java diretamente em suas páginas HTML, em contraste com os servlets, nos quais você coloca HTML dentro do código Java. JSP é parte do Java 2 Enterprise Edition (J2EE).

JSP permite que você separe o conteúdo dinâmico de uma página Web de sua apresentação. Ele satisfaz dois tipos diferentes de desenvolvedores: desenvolvedores HTML, que são responsáveis pelo projeto gráfico da página, e desenvolvedores Java, que tratam do desenvolvimento de software que criará o conteúdo dinâmico.

Para os amantes do XML, boas notícias: as páginas JSP podem ser escritas em XML bem-formado através da utilização de XSLT. (Isto significa que JSP pode fazer uso total das características do XML e de todas as ferramentas e suporte que existem para ele.)

Como JSP é parte do J2EE padrão, você pode desenvolver JPSs para várias plataformas. Além disso, diversas empresas e desenvolvedores de aplicações podem fornecer componentes JavaBean e definir tags JSP personalizadas que podem ser referenciadas de uma página JSP.

As Vantagens do JSP

JSP possui várias vantagens sobre a maioria de seus "concorrentes". Aqui estão algumas delas

Versus Active Server Pages (ASP)

ASP é uma tecnologia concorrente da Microsoft. As vantagens do JSP são duas. Primeiro, a parte dinâmica é escrita em Java, e não VBScript ou outra linguagem específica do ASP, sendo assim mais poderosa e mais adaptada para aplicações complexas que requerem componentes reusáveis. Segundo, JSP é portável para outros sistemas operacionais e servidores Web; você não fica preso ao Windows NT/2000/XP e IIS. Você poderia usar o mesmo argumento ao comparar JSP com ColdFusion; com JSP você pode usar Java e não está amarrado a um produto para servidor particular.

Versus PHP

PHP é uma linguagem de scripting embutida em HTML gratuita e de código aberto que é um pouco similar tanto ao ASP quanto ao JSP. A vantagem do JSP é que a parte dinâmica é escrita em Java, que você provavelmente já conhece, que já tem uma API abrangente para rede, acesso a banco de dados, objetos distribuídos, e outras facilidades, enquanto PHP requer a aprendizagem de toda uma nova linguagem.

Versus Servlets Puros

JSP não fornece capacidades que não pudessem a princípio ser obtidas com um servlet. De fato, "por baixo do pano" documentos JSP são traduzidos automaticamente em servlets. Mas é mais conveniente escrever (e modificar!) HTML tradicional do que ter um zilhão de instruções println que geram o HTML. Além disso, separando a apresentação do conteúdo, você pode colocar pessoas diferentes em tarefas diferentes: seu web designer pode construir o HTML usando ferramentas familiares e deixar espaços para seus programadores de servlets inserirem conteúdo dinâmico.

Versus Server-Side Includes (SSI)

SSI é uma tecnologia bastante usada para inserir peças definidas externamente dentro de uma página Web estática. JSP é melhor porque você tem um rico conjunto de ferramentas para construir estas peças externas e tem mais opções com relação ao estágio da resposta HTTP na qual esta peça realmente está inserida. Além disso, SSI é realmente voltado apenas para inclusões simples, e não para programas "reais" que usam dados, fazem conexões a bancos de dados, e outras coisas do tipo.

Versus JavaScript

JavaScript, que é completamente diferente da linguagem de programação Java, é geralmente usado para gerar HTML dinâmico no cliente, construir partes da página web a medida que o browser carrega o documento. Esta é uma capacidade útil mas trata apenas situações onde a informação dinâmica está baseada no ambiente do cliente. Com a exceção de cookies, os dados de uma requisição HTTP não estão disponíveis para rotinas JavaScript do lado do cliente. E, uma vez que JavaScript carece de rotinas para programação em rede, o código JavaScript no cliente não pode acessar recursos do lado do servidor como bancos de dados, catálogos, etc. JavaScript também pode ser usado no servidor, mais notavelmente em servidores Netscape e como uma linguagem de scripting para IIS. Java é muito mais poderoso, flexível, confiável e portável.

Versus HTML Estático

HTML regular, é claro, não contém informação dinâmica, assim páginas HTML estáticas não podem se basear em entradas dos usuários ou em fontes de dados no servidor.
